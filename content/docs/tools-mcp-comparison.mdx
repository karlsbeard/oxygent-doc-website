### Use Cases

#### StdioMCPClient

**âœ… Best For:**
- Local MCP servers (Node.js, Python, etc.)
- Development and testing
- NPX-based MCP packages
- Command-line tools with MCP interface
- Single-machine deployments
- Lowest latency requirements

**âŒ Not Suitable For:**
- Remote MCP servers
- Web-based deployments
- Services requiring authentication
- Multi-machine architectures

**Example Use Cases:**
```python

oxy.StdioMCPClient(
    name="filesystem",
    params={
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"]
    }
)

# Custom local Python server
oxy.StdioMCPClient(
    name="local_tools",
    params={
        "command": "uv",
        "args": ["--directory", "./mcp_servers", "run", "my_server.py"]
    }
)
```

#### SSEMCPClient

**âœ… Best For:**
- Remote MCP servers over HTTP
- Real-time unidirectional streaming
- Simple authentication (headers)
- Cloud-hosted MCP services
- Web application integrations
- Scalable multi-client setups

**âŒ Not Suitable For:**
- Local-only tools
- Complex bidirectional streaming
- Use cases requiring clientâ†’server streaming

**Example Use Cases:**

#### StreamableMCPClient

**âœ… Best For:**

## Common Scenarios

### Scenario 1: Local Development

**Recommendation:** StdioMCPClient

**Why:**
- Fastest latency (no network)
- Simple setup
- Easy debugging (process logs)
- Works offline

**Example:**
```python
oxy_space = [
    oxy.HttpLLM(name="llm", ...),
    oxy.StdioMCPClient(
        name="dev_tools",
        params={
            "command": "npx",
            "args": ["-y", "@modelcontextprotocol/server-filesystem", "./workspace"]
        }
    ),
    oxy.ReActAgent(name="agent", llm_model="llm", tools=["dev_tools"])
]
```

### Scenario 2: Production Web Service

**Recommendation:** SSEMCPClient or StreamableMCPClient

**Why:**
- Remote access required
- Authentication needed
- Scalable architecture
- Multiple clients

**Example (SSE for simple streaming):**
```python
oxy_space = [
    oxy.HttpLLM(name="llm", ...),
    oxy.SSEMCPClient(
        name="prod_tools",
        sse_url="https://api.example.com/mcp/sse",
        headers={"Authorization": f"Bearer {token}"},
        is_keep_alive=True
    ),
    oxy.ReActAgent(name="agent", llm_model="llm", tools=["prod_tools"])
]
```

**Example (Streamable for complex streaming):**
```python
oxy_space = [
    oxy.HttpLLM(name="llm", ...),
    oxy.StreamableMCPClient(
        name="prod_streaming",
        server_url="https://api.example.com/mcp",
        headers={"Authorization": f"Bearer {token}"},
        is_keep_alive=True
    ),
    oxy.ReActAgent(name="agent", llm_model="llm", tools=["prod_streaming"])
]
```

### Scenario 3: Multi-Environment Setup

**Recommendation:** Mix of all clients

**Why:**
- Different environments have different needs
- Local tools for dev
- Remote APIs for production

**Example:**
```python
oxy_space = [
    oxy.HttpLLM(name="llm", ...),

    # Local tools (fastest)
    oxy.StdioMCPClient(
        name="local_fs",
        params={"command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]}
    ),

    # Remote API (SSE)
    oxy.SSEMCPClient(
        name="remote_api",
        sse_url="https://api.example.com/sse",
        headers={"Authorization": f"Bearer {token}"}
    ),

    # Streaming service (Streamable)
    oxy.StreamableMCPClient(
        name="streaming_service",
        server_url="https://stream.example.com/mcp",
        headers={"Authorization": f"Bearer {token}"}
    ),

    oxy.ReActAgent(
        name="agent",
        llm_model="llm",
        tools=["local_fs", "remote_api", "streaming_service"]
    )
]
```

### Scenario 4: Dynamic Authentication

**Recommendation:** SSEMCPClient or StreamableMCPClient with `is_dynamic_headers=True`

**Why:**
- Per-request authentication
- User-specific tokens
- Request-level headers

**Example:**
```python
oxy.SSEMCPClient(
    name="dynamic_auth",
    sse_url="https://api.example.com/sse",
    headers={"X-Service-Key": "base-key"},
    is_dynamic_headers=True,
    is_inherit_headers=True,
    is_keep_alive=False  # Required
)

# Later, provide user-specific token
await mas.call(
    callee="agent",
    arguments={
        "messages": [...],
        "headers": {"Authorization": f"Bearer {user_token}"}
    }
)
```

## Migration Guide

### From Stdio to SSE

**Before (Local):**
```python
oxy.StdioMCPClient(
    name="tools",
    params={
        "command": "python",
        "args": ["./mcp_server.py"]
    }
)
```

**After (Remote):**
```python
# Start server separately: python mcp_server_sse.py
oxy.SSEMCPClient(
    name="tools",
    sse_url="http://localhost:9000/sse"
)
```

### From SSE to Streamable

**Before (SSE):**
```python
oxy.SSEMCPClient(
    name="tools",
    sse_url="https://api.example.com/sse",
    headers={"Authorization": f"Bearer {token}"}
)
```

**After (Streamable):**
```python
oxy.StreamableMCPClient(
    name="tools",
    server_url="https://api.example.com/mcp",  # Note: different endpoint
    headers={"Authorization": f"Bearer {token}"}
)
```

## Best Practices

### For StdioMCPClient

1. **Use absolute paths** for directory-based commands
2. **Set environment variables** for configuration
3. **Handle process cleanup** properly
4. **Monitor process logs** for debugging

### For SSEMCPClient

1. **Use `is_keep_alive=True`** for production
2. **Add retry logic** for network failures
3. **Monitor connection health**
4. **Use dynamic headers** only when necessary

### For StreamableMCPClient

1. **Optimize for your workload** (persistent vs transient)
2. **Tune concurrency** via `semaphore`
3. **Set appropriate timeouts** for long streams
4. **Use middlewares** for monitoring

## Troubleshooting

### StdioMCPClient Issues

**Process won't start:**
- Check command is in PATH
- Verify file permissions
- Check directory exists

**Tools not discovered:**
- Verify MCP server implements protocol correctly
- Check server logs for errors

### SSEMCPClient Issues

**Connection refused:**
- Verify server is running
- Check URL and port
- Test with curl/browser

**Authentication failed:**
- Verify token is valid
- Check header format

### StreamableMCPClient Issues

**Streaming timeout:**
- Increase `timeout` parameter
- Check network latency
- Optimize server response
