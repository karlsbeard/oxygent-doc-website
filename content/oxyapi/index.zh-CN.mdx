## 代理

代理是 OxyGent 框架的核心组件，负责执行特定任务和做出决策。

- **[ChatAgent API](/oxyapi/agents-chat-api)** - 基于聊天的代理 API 文档
- **[ParallelAgent API](/oxyapi/agents-parallel-api)** - 并行代理 API 文档
- **[RAGAgent API](/oxyapi/agents-rag-api)** - 检索增强生成代理 API 文档
- **[ReactAgent API](/oxyapi/agents-react-api)** - 基于 ReAct 模式的代理 API 文档
- **[SSEAgent API](/oxyapi/agents-sse-agent-api)** - 服务器发送事件代理 API 文档
- **[WorkflowAgent API](/oxyapi/agents-workflow-api)** - 工作流代理 API 文档

## 快速开始

如果您是 OxyGent API 的新手，我们建议从以下步骤开始：

1. 查看 [ReactAgent API](/oxyapi/agents-react-api) 以了解基本的代理使用方法
2. 学习 [HttpLLM API](/oxyapi/llms-http-api) 以了解如何配置大语言模型
3. 探索 [FunctionHub API](/oxyapi/tools-function-hub-api) 以了解工具系统

## API 约定

所有 OxyGent API 都遵循以下约定：

- **异步操作**：大多数 API 方法都是异步的，需要使用 `await` 关键字
- **类型安全**：所有 API 都提供完整的类型注释
- **错误处理**：API 方法会抛出特定的异常类型以便于错误处理
- **灵活配置**：支持多种配置方法，包括参数传递和配置文件
