## 类定义

```python
from oxygent import oxy

# 创建 StreamableMCPClient 实例
client = oxy.StreamableMCPClient(
    name="streaming_tools",
    server_url="http://127.0.0.1:8000/mcp",
    headers={"Authorization": "Bearer token"},
    middlewares=[]
)
```

## 构造函数参数

### 必需参数

|参数|类型|必需|默认值|描述|
|-----------|------|----------|---------|-------------|
| `name` | `str` | **是** | - | MCP 客户端的唯一标识符。代理在其 `tools` 列表中引用此名称 |
| `server_url` | `AnyUrl` | **是** | `""` | 流式 MCP 服务器的 HTTP(S) 端点 URL |

### 可选参数

|参数|类型|默认值|描述|
|-----------|------|---------|-------------|
| `middlewares` | `list[Any]` | `[]` | 用于流处理的客户端 MCP 中间件列表 |

### 从 BaseMCPClient 继承

|参数|类型|默认值|描述|
|-----------|------|---------|-------------|
| `headers` | `dict[str, str]` | `{}` | 随所有请求发送的 HTTP 头 |
| `is_dynamic_headers` | `bool` | `False` | 启用每个请求的动态头更新 |
| `is_inherit_headers` | `bool` | `False` | 从 OxyRequest._headers 继承头 |
| `is_keep_alive` | `bool` | 配置默认值 | 在调用之间保持 HTTP 连接活动 |

### 从 BaseTool 继承

|参数|类型|默认值|描述|
|-----------|------|---------|-------------|
| `timeout` | `float` | `60` | 每次工具调用的最大执行时间（秒） |
| `semaphore` | `int` | `16` | 最大并发工具执行数 |
| `retries` | `int` | `2` | 失败时的重试次数 |
| `delay` | `float` | `1.0` | 重试尝试之间的延迟（秒） |
| `is_permission_required` | `bool` | `True` | 代理是否需要权限来调用工具 |
| `is_save_data` | `bool` | `True` | 是否保存执行跟踪 |

### 内部属性

|属性|类型|描述|
|-----------|------|-------------|
| `included_tool_name_list` | `list[str]` | 从 MCP 服务器发现的工具名称列表 |
| `_session` | `ClientSession` | MCP 协议会话（在 `init()` 期间初始化） |
| `_http_transport` | `tuple` | HTTP 传输元组（read、write、cancel）用于持久连接 |
| `_exit_stack` | `AsyncExitStack` | 用于清理的资源管理上下文 |

## 参数详情

### name

- **类型**: `str`
- **必需**: 是
- **描述**: 在多智能体系统中此流式 MCP 客户端的唯一标识符。智能体在其 `tools` 列表中引用此名称，以使用此流式服务器提供的所有工具。

**Example:**
```python
oxy.StreamableMCPClient(
    name="streaming_api_tools",  # 智能体在 tools 列表中使用 "streaming_api_tools"
    ...
)
```

### server_url

- **类型**: `AnyUrl` (Pydantic URL 类型)
- **必需**: 是
- **描述**: 流式 MCP 服务器托管的 HTTP(S) URL 端点。此 URL 指向服务器的流式端点。

**格式：**
- 必须是有效的 URL
- 通常以 `/mcp` 或类似路径结尾
- 可以是 HTTP（本地/开发）或 HTTPS（生产环境）
- 与 SSE 端点不同（没有 `/sse` 后缀）

**Examples:**
```python
# 本地开发
server_url="http://localhost:8000/mcp"
server_url="http://127.0.0.1:3000/api/mcp"

# 生产环境
server_url="https://api.example.com/mcp"
server_url="https://streaming.myapp.com/mcp/v1"
```

**URL 处理：**
URL 通过 `build_url()` 工具处理，该工具：
- 确保格式正确
- 处理尾部斜杠
- 验证 URL 结构

### headers

- **类型**: `dict[str, str]`
- **默认**: `{}`
- **描述**: 包含在所有流式请求中的 HTTP 头。通常用于身份验证、版本控制和自定义元数据。

**Common Use Cases:**

1. **Authentication:**
```python
headers={
    "Authorization": "Bearer your-streaming-token",
    "X-API-Key": os.getenv("STREAMING_API_KEY")
}
```

2. **API Configuration:**
```python
headers={
    "X-API-Version": "v2",
    "Accept": "application/json",
    "X-Stream-Mode": "bidirectional"
}
```

3. **Client Identification:**
```python
headers={
    "X-Client-ID": "oxygent-streaming",
    "X-Application": "my-app",
    "User-Agent": "OxyGent/1.0"
}
```

**行为：**
- 头信息随每个 HTTP 流式连接发送
- 如果 `is_dynamic_headers=True`，可以按请求覆盖
- 如果继承，`host` 头会自动移除

### is_dynamic_headers

- **类型**: `bool`
- **默认**: `False`
- **描述**: 启用按请求动态更新头信息。启用时，可以在工具调用中提供头信息，并将覆盖/合并基础头信息。

**要求：**
- 动态头信息必须设置 `is_keep_alive=False`
- 为每个工具调用创建新的 HTTP 连接

**Example:**
```python
# 启用动态头信息
oxy.StreamableMCPClient(
    name="dynamic_streaming",
    server_url="http://localhost:8000/mcp",
    headers={
        "X-Service-Key": "default"  # 基础头信息
    },
    is_dynamic_headers=True,
    is_keep_alive=False  # 必需
)

# 在工具调用中提供动态头信息
await oxy_request.call(
    callee="tool_name",
    arguments={"param": "value"},
    headers={  # 覆盖/合并基础头信息
        "X-User-ID": "user123",
        "X-Request-ID": "req456"
    }
)
```

### is_inherit_headers

- **类型**: `bool`
- **默认**: `False`
- **描述**: 从 OxyRequest 上下文（`_headers` 字段）继承头信息。用于在调用链中传播头信息。

**行为：**
- 如果 `is_inherit_headers=True`，读取 `oxy_request.shared_data.get("_headers", {})`
- 自动移除 `host` 头
- 合并：`基础头信息 | 继承头信息 | 动态头信息`

**Example:**
```python
oxy.StreamableMCPClient(
    name="inherit_streaming",
    server_url="http://localhost:8000/mcp",
    is_dynamic_headers=True,
    is_inherit_headers=True,  # 从上下文继承
    is_keep_alive=False
)

# 上游提供头信息
await mas.call(
    callee="agent",
    arguments={
        "messages": [...],
        "_headers": {  # 由流式客户端继承
            "X-Trace-ID": "trace123",
            "X-Session": "session456"
        }
    }
)
```

### is_keep_alive

- **类型**: `bool`
- **默认**: 来自 `Config.get_tool_mcp_is_keep_alive()`
- **描述**: 控制 HTTP 流式连接生命周期。

**True（持久连接）：**
- 在 `init()` 期间建立单个 HTTP 连接
- 连接在所有工具调用中重用
- 更低延迟（无重连开销）
- 工具在启动时发现一次
- 不能使用动态头信息
- **推荐用于生产环境**

**False（临时连接）：**
- 每个工具调用创建新的 HTTP 连接
- 每次都是新状态
- 动态头信息必需
- 更高延迟（重连成本）
- 每次重新发现工具

**Examples:**
```python
# 持久连接（推荐）
oxy.StreamableMCPClient(
    name="persistent",
    server_url="http://localhost:8000/mcp",
    is_keep_alive=True  # 默认
)

# 临时连接（用于动态头信息）
oxy.StreamableMCPClient(
    name="transient",
    server_url="http://localhost:8000/mcp",
    is_dynamic_headers=True,
    is_keep_alive=False  # 必需
)
```

### middlewares

- **类型**: `list[Any]`
- **默认**: `[]`
- **描述**: 用于处理流式请求和响应的客户端中间件对象列表。中间件拦截和修改流数据。

**中间件接口：**
中间件对象应实现流处理方法：
- `process_request(request)` - 发送前调用
- `process_response(response)` - 接收后调用
- `process_stream(stream)` - 流式传输期间调用

**Example:**
```python
class StreamMetricsMiddleware:
    def __init__(self):
        self.request_count = 0
        self.bytes_sent = 0
        self.bytes_received = 0

    async def process_stream(self, stream):
        self.request_count += 1
        # 处理流
        return stream

class StreamLoggingMiddleware:
    async def process_request(self, request):
        print(f"[STREAM REQUEST] {request}")
        return request

    async def process_response(self, response):
        print(f"[STREAM RESPONSE] {response}")
        return response

oxy.StreamableMCPClient(
    name="middleware_streaming",
    server_url="http://localhost:8000/mcp",
    middlewares=[
        StreamMetricsMiddleware(),
        StreamLoggingMiddleware()
    ]
)
```

**兼容性：**
- 依赖于 MCP 客户端库版本
- 如果 `session.add_middleware()` 不可用，记录警告
- 不兼容版本中忽略中间件

### timeout

- **类型**: `float`
- **默认**: `60`
- **描述**: 每个流式工具调用的最大执行时间（秒）。包括连接、流式传输和响应时间。

**指导原则：**
- 快速流：10-30 秒
- 标准操作：30-60 秒
- 长时间运行的流：60-300 秒
- 非常长的进程：300+ 秒

**Example:**
```python
oxy.StreamableMCPClient(
    name="long_streaming",
    server_url="http://api.example.com/mcp",
    timeout=300  # 长流 5 分钟
)
```

## 方法

### init()

初始化到 MCP 服务器的 HTTP 流式连接。

```python
async def init(self, is_fetch_tools: bool = True) -> None
```

**参数:**
- `is_fetch_tools` (bool): 初始化后是否自动发现和注册工具。默认: `True`

**行为：**

取决于 `is_dynamic_headers` 和 `is_keep_alive`：

**情况 1：持久连接（`is_keep_alive=True` 且 `not is_dynamic_headers`）**
1. 使用 `build_url(server_url)` 构建服务器 URL
2. 建立流式 HTTP 客户端连接
3. 解包传输元组：`(read, write, cancel)`
4. 使用读/写流创建 MCP `ClientSession`
5. 将所有 `middlewares` 添加到会话
6. 初始化 MCP 协议
7. 如果 `is_fetch_tools=True`，发现工具
8. 存储 `_http_transport` 以供重用

**情况 2：临时连接（`is_keep_alive=False` 或 `is_dynamic_headers=True`）**
1. 创建临时流式 HTTP 连接
2. 通过 `list_tools()` 发现工具
3. 立即关闭连接
4. 后续调用创建新连接

**错误处理:**
- 失败时记录包含服务器名称的错误
- 自动调用 `cleanup()`
- 抛出包含"Server \{name\} error"的 `Exception`

**示例（内部使用）：**
```python
# 由 MAS 自动调用
async with MAS(oxy_space=[client, ...]) as mas:
    # client.init() 在此处调用
    ...
```

### call_tool()

在流式 MCP 服务器上执行工具。

```python
async def call_tool(
    self,
    tool_name: str,
    arguments: dict,
    headers: dict[str, str] = None
) -> Any
```

**参数:**
- `tool_name` (str): 要调用的工具名称
- `arguments` (dict): 工具特定的参数
- `headers` (dict, optional): 为此调用覆盖/合并的头

**返回：** 包含工具执行结果的 MCP 响应对象

**行为：**
1. 使用 `build_url(server_url)` 构建 URL
2. 使用头建立流式 HTTP 客户端
3. 解包传输：`(read, write, cancel)`
4. 创建临时 ClientSession
5. 初始化 MCP 协议
6. 使用参数调用工具
7. 返回响应
8. 自动关闭连接（异步上下文）

**注意：** 当 `is_keep_alive=False` 或 `is_dynamic_headers=True` 时使用。对于持久连接，`_execute()` 使用现有会话。

**示例（内部使用）：**
```python
# 为临时连接内部调用
response = await client.call_tool(
    tool_name="stream_process",
    arguments={"data": "large_dataset"},
    headers={"X-Priority": "high"}
)
```

### list_tools()

从流式 MCP 服务器发现并注册工具。

```python
async def list_tools(self) -> None
```

**行为：**
1. 检查 `_session` 是否已初始化（持久模式）
2. 通过 MCP 协议调用 `_session.list_tools()`
3. 调用 `add_tools()` 创建 `MCPTool` 包装器
4. 在 MAS 中注册每个工具

**抛出：**
- 如果服务器未初始化（持久模式），抛出 `RuntimeError`

**注意：** 在临时模式下，在 `init()` 期间使用临时会话调用。

**示例（内部使用）：**
```python
# 在 init() 期间自动调用
await client.list_tools()
```

### add_tools()

为发现的工具创建 MCPTool 包装器。

```python
def add_tools(self, tools_response) -> None
```

**参数:**
- `tools_response`: 来自 `list_tools()` 的响应，包含工具定义

**行为：**
1. 提取排除敏感字段的模型转储:
   - `server_url`
   - `sse_url`
   - `headers`
   - `middlewares`
   - `included_tool_name_list`
   - `name`, `desc`, `mcp_client`, `server_name`, `input_schema`
2. 遍历响应中的工具
3. 为每个工具创建 `MCPTool` 实例，包含:
   - 工具名称和描述
   - 对此流式客户端的引用
   - 输入模式
   - 钩子函数（如果已配置）
4. 设置 MAS 引用并注册
5. 添加到 `included_tool_name_list`

**示例（内部使用）：**
```python
# 由 list_tools() 调用
tools_response = await session.list_tools()
client.add_tools(tools_response)
```

### cleanup()

清理 HTTP 流式资源和连接。

```python
async def cleanup(self) -> None
```

**行为：**
1. 获取清理锁（防止并发清理）
2. 关闭 `AsyncExitStack`（终止 HTTP 连接和会话）
3. 优雅地处理 `asyncio.CancelledError`
4. 抑制其他异常
5. 将 `_session` 和 `_stdio_context` 设置为 `None`

**何时调用：**
- MAS 退出时自动调用
- 初始化错误时自动调用
- 如果需要可以手动调用

**示例（通常自动）：**
```python
# 自动清理
async with MAS(oxy_space=[client, ...]) as mas:
    ...
# cleanup() 在此处调用
```

## 使用示例

### 基础流式传输

```python
from oxygent import MAS, oxy
import os

client = oxy.StreamableMCPClient(
    name="basic_streaming",
    server_url="http://localhost:8000/mcp"
)

oxy_space = [
    oxy.HttpLLM(name="llm", ...),
    client,
    oxy.ReActAgent(
        name="agent",
        llm_model="llm",
        tools=["basic_streaming"]
    )
]
```

### 使用身份验证

```python
secure_client = oxy.StreamableMCPClient(
    name="secure_streaming",

        "Authorization": f"Bearer {os.getenv('STREAMING_TOKEN')}",
        "X-API-Version": "v2"
```

### 使用动态头

```python
dynamic_client = oxy.StreamableMCPClient(

    is_dynamic_headers=True,
    is_inherit_headers=True,
```

### 使用中间件

```python
class RateLimitMiddleware:
    def __init__(self, max_requests=100):
        self.max_requests = max_requests
        self.request_count = 0

    async def process_request(self, request):
        if self.request_count >= self.max_requests:
            raise Exception("Rate limit exceeded")
        self.request_count += 1
        return request

middleware_client = oxy.StreamableMCPClient(
    name="rate_limited_streaming",
    server_url="http://localhost:8000/mcp",
    middlewares=[RateLimitMiddleware(max_requests=1000)]
)
```

### 高性能配置

```python
perf_client = oxy.StreamableMCPClient(
    name="high_perf_streaming",
    server_url="http://localhost:8000/mcp",
    is_keep_alive=True,          # 持久连接
    timeout=120,                 # 2 分钟超时
    semaphore=64,                # 64 个并发流
    retries=1,                   # 快速失败
    is_permission_required=False # 无权限开销
)
```

## 错误处理

### 连接错误

```python
try:
    client = oxy.StreamableMCPClient(
        name="unavailable",
        server_url="http://localhost:9999/mcp"  # 未运行
    )
    async with MAS(oxy_space=[client, ...]) as mas:
        await mas.call(...)
except Exception as e:
    print(f"Connection error: {e}")
```

### 超时错误

```python
client = oxy.StreamableMCPClient(
    name="strict_timeout",
    server_url="http://slow-server.com/mcp",
    timeout=10  # 非常短的超时
)
# 超过 10 秒的流将引发超时
```

## 性能考虑

### 连接模式

|Aspect|Persistent|Transient|
|--------|-----------|-----------|
| **延迟** | 很低 | 高 |
| **吞吐量** | 很高 | 较低 |
| **资源使用** | 较高 | 较低 |
| **动态头信息** | 否 | 是 |
| **推荐用于** | 生产环境 | 开发/测试 |

### 并发优化

```python
# 高吞吐量
oxy.StreamableMCPClient(
    name="throughput",
    server_url="http://localhost:8000/mcp",
    semaphore=128,  # 多个并发流
    timeout=60
)

# 低延迟
oxy.StreamableMCPClient(
    name="latency",
    server_url="http://localhost:8000/mcp",
    semaphore=16,
    timeout=5,
    retries=0  # 最快响应无重试
)
```
